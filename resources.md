# Resources

## Definitions

### Web Accessibility
Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with the web. 

### WCAG 2.2
The Web Content Accessibility Guidelines (WCAG) 2.2 provide a set of recommendations for making web content more accessible to people with disabilities.

## Audit Methodology
1. **Preparation**  
   Understand the target user group and their needs.
2. **Accessibility Evaluation**  
   Conduct manual testing and use automated tools to evaluate compliance with WCAG 2.2 guidelines.
3. **User Testing**  
   Involve users with disabilities to obtain their feedback on the accessibility of the content.
4. **Reporting**  
   Document findings and create a plan for remediation of any identified issues.

## References to WCAG 2.2 Guidelines
- **Perceivable**: Information and user interface components must be presentable to users in ways they can perceive.  
Reference: [WCAG 2.2 Guidelines - Perceivable](https://www.w3.org/WAI/WCAG22/)

- **Operable**: User interface components and navigation must be operable.  
Reference: [WCAG 2.2 Guidelines - Operable](https://www.w3.org/WAI/WCAG22/)

- **Understandable**: Information and operation of user interface must be understandable.  
Reference: [WCAG 2.2 Guidelines - Understandable](https://www.w3.org/WAI/WCAG22/)

- **Robust**: Content must be robust enough to be interpreted by a wide variety of user agents, including assistive technologies.
Reference: [WCAG 2.2 Guidelines - Robust](https://www.w3.org/WAI/WCAG22/)
